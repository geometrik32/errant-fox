# Errant Fox API (WebSocket)

Взаимодействие с сервером происходит через WebSocket. Все сообщения упакованы в формат **Protobuf** (`ClientToServerCmd`).

## 1. Подключение
*   **URL**: `ws://<server_ip>:<port>/ws`
*   **Авторизация**: JWT токен должен быть передан при логине.

## 2. Основные команды (ClientToServerCmd)

| Команда | Описание | Входные параметры |
| :--- | :--- | :--- |
| `login` | Вход в систему | `username`, `password` |
| `openMediaFile` | Открыть видео | `mediaFileId` |
| `addComment` | Добавить комментарий | `mediaFileId`, `comment`, `timecode`, `drawing` |
| `hemaAddBout` | Добавить сход | `bout` (объект HemaBout) |
| `hemaListUsers` | Список бойцов | - |

## 3. События сервера (ServerToClientCmd)

| Событие | Описание | Когда происходит |
| :--- | :--- | :--- |
| `loginResult` | Результат логина | Ответ на `login` |
| `openMediaFile` | Данные видео | Ответ на `openMediaFile` |
| `hemaMediaBouts`| Список сходов | При открытии видео или обновлении схода |
| `showMessages` | Уведомления | Ошибки или подтверждение действий |

## 4. Особенности
*   **ID участников**: Теперь используются строковые логины (например, "ivanov_i").
*   **Синхронизация**: Если вы добавили комментарий, сервер сам разошлет событие `addComments` всем, кто смотрит это же видео.
*   **Таймкоды**: Все таймкоды передаются в секундах (float).
