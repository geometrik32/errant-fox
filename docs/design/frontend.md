# Директива: Дизайн Frontend (Errant Fox)

## Цель
Полный редизайн интерфейса приложения Errant Fox с использованием стиля (мягкие тени, скругления, glassmorphism, желтые акценты). Реализация переключения между светлой и темной темой, а также полная адаптация под мобильные устройства.

## Входные данные
- Существующие компоненты Svelte 5 (runes) в `frontend/src/` (функционал компонентов не изменяется).
- Стандартный стек: HTML, CSS (переменные), Svelte 5.

## Требования к стилю (Design Tokens)
1. **Шрифты**: Использовать `Inter` (Google Fonts).
2. **Формы**: Закругленные углы для карточек (`border-radius: 20px` или `24px`) и кнопок (`border-radius: 9999px`).
3. **Эффекты**: Использование `backdrop-filter: blur(10px)` для эффекта стекла, мягкие тени `box-shadow: 0 8px 32px rgba(0,0,0,0.05)`.
4. **Светлая тема (default)**:
   - Фон: градиент от белого/кремового к светло-желтому.
   - Текст: темно-серый/черный.
   - Карточки: белые с прозрачностью.
5. **Темная тема**:
   - Фон: градиент темно-серый/черный.
   - Текст: светло-серый/белый.
   - Карточки: темно-серые с прозрачностью.
6. **Адаптивность**: При `max-width: 768px` элементы выстраиваются в одну колонку (flex-direction: column), скрываются боковые панели в гамбургер-меню (если применимо), уменьшаются отступы.

## Инструменты
- Использование CSS переменных (`var(--bg-gradient)`, `var(--card-bg)`, и т.д.) в [`app.css`](frontend/src/app.css).
- Обертывание логики переключения в `data-theme="dark"` атрибут на теге `<html>` или `<body>`.
- Переключение темы — в [`Header.svelte`](frontend/src/lib/ui/Header.svelte).

## Выходные данные
- Обновленные файлы `*.svelte` с новыми CSS-стилями.
- Глобальный файл [`app.css`](frontend/src/app.css) с дизайн-токенами.
- Обновленный [`index.html`](frontend/index.html).

## Edge Cases (Особые случаи)
- Компоненты, которые генерируются динамически (графики Chart.js), должны наследовать цвета темы через CSS-переменные.
- При переключении темы, графики могут потребовать перерисовки (вызов `.update()`), либо использования универсальных цветов, которые хорошо выглядят на обоих фонах.
- SVG-компоненты ([`BodySilhouette.svelte`](frontend/src/lib/stats/BodySilhouette.svelte), [`HitZonePicker.svelte`](frontend/src/lib/player/HitZonePicker.svelte)) используют `currentColor` и CSS-переменные для совместимости с темами.
- Цвета бойцов генерируются динамически через [`resolveColor()`](frontend/src/lib/api/types.ts:19) и должны быть читаемы на обоих фонах темы.
