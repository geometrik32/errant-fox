# Техническое задание: Errant Fox

## 1. Обзор системы

Веб-приложение для видеоанализа тренировочных спаррингов по HEMA (длинный меч).
Одна установка на домашнем сервере TrueNAS. Все видео хранятся в Seafile.

---

## 2. Пользователи и права

| Возможность | Пользователь | Admin |
|---|---|---|
| Просмотр видео, статистики, чата | ✓ | ✓ |
| Разметка сходов | ✓ | ✓ |
| Комментарии в чате | ✓ | ✓ |
| Редактирование своего профиля | ✓ | ✓ |
| Создание пользователей | — | ✓ |
| Управление списком техник | — | ✓ |

**Профиль**: имя, аватар, пароль — пользователь меняет сам. Статус Admin нельзя.
**Admin при создании**: логин, пароль, имя, аватар (опционально).

---

## 3. Интеграция с Seafile

- Параметры подключения (URL, API-токен) задаются один раз в конфиге при деплое.
- Приложение автоматически индексирует новые видео из Seafile без участия пользователя.
- Структура: папки по датам тренировок → дата парсится из имени папки. Внутри — видеофайлы с произвольными именами (имена не используются нигде в UI).
- Видео стримится напрямую из Seafile, не кешируется на сервере.
- Статичные превью-кадры для галереи — через Seafile API.

---

## 4. Модель данных

### Fighter
`id`, `username`, `password_hash`, `display_name`, `avatar_url`, `is_admin`, `created_at`

### Technique
`id`, `name` — глобальный список, управляется Admin (добавить / удалить)

### HitZone (фиксированный enum)
Голова, Тело, Рука левая, Рука правая, Нога левая, Нога правая

### Video
`id`, `seafile_path`, `fighter_a_id` (nullable), `fighter_b_id` (nullable), `date`, `created_at`

Итоговый счёт — вычисляется динамически как сумма `score_a` и `score_b` всех сходов.

### Bout (Сход)
`id`, `video_id`, `time_start_ms`, `time_end_ms`, `order_index`

Боец A: `score_a` (int), `technique_a_id` (nullable), `hit_zone_a` (nullable), `result_a` (hit / miss / blocked)
Боец B: `score_b` (int), `technique_b_id` (nullable), `hit_zone_b` (nullable), `result_b` (hit / miss / blocked)

> Аналитика: статистика полученных ударов бойца A строится из данных бойца B в сходах, где участвовал A (и наоборот).

### Comment
`id`, `video_id`, `author_id`, `timestamp_ms`, `text`, `reply_to_id`, `created_at`

## 5. Экраны

### 5.1. Авторизация
Логотип, заголовок "Sign In", поля Username / Password, кнопка "Sign In".
Фон — тёмно-синий градиент. Регистрация отсутствует.

---

### 5.2. Глобальный Header
- **Лево**: логотип + "Errant Fox" → клик → галерея.
- **Центр**: переключатель `[Видео] [Бойцы]`.
- **Право**: иконка настроек + аватар → dropdown: профиль, выход; для Admin дополнительно: создать бойца, управление техниками.

---

### 5.3. Экран «Видео» — Галерея

**Сайдбар (слева)**:
- Список чекбоксов бойцов с количеством видео. Мультивыбор.
- Виджет выбора диапазона дат.

**Сетка карточек (центр)**:
- Превью 16:9 (статичный кадр). При наведении — анимация scrub по превью-кадрам.
- **Размеченное видео**: `Боец A vs Боец B`, счёт `[X : Y]`, дата.
- **Неразмеченное видео**: на месте имён и счёта — визуальный маркер (иконка + "Заполните данные").
- Клик на неразмеченное → модальная форма: выбор Бойца A и Бойца B (дата уже проставлена из имени папки Seafile).
- Клик на размеченное → экран просмотра видео.

---

### 5.4. Экран «Бойцы» — Статистика

**Сайдбар (слева)**:
- Список имён бойцов для переключения дашбордов.

> Единственный фильтр данных — через таблицу "История боёв" (фильтры по столбцам). Все графики и силуэты реагируют на него.

**Дашборд бойца**:

**Header**: аватар + имя + дата первого зафиксированного боя.

**Блоки быстрой статистики**:
- Использую приём чаще всего: `[название техники]`
- Промахиваюсь чаще всего с приёмом: `[название техники]`
- Получаю урон чаще всего от приёма: `[название техники]`

**График 1 — Частота поединков**: ось X — недели, ось Y — количество боёв.

**График 2 — Динамика результатов**:
- Ось X — бои по хронологии, ось Y — победа вверх / поражение вниз (бинарно).
- Фильтр по конкретному противнику (дропдаун над графиком).
- В заголовке: общий процент побед и поражений.
- Победитель — у кого сумма очков за все сходы видео больше.

**График 3 — Прогресс по баллам**: линейный график суммарных очков за тренировку по хронологии.

**Зона поражения — нанесённый урон**: силуэт человека в T-позе, интенсивность закраски зон по частоте попаданий бойца.

**Зона поражения — полученный урон**: то же, но по ударам, пришедшим по бойцу.

> Оба силуэта динамически обновляются при фильтрации таблицы истории боёв.

**История боёв** (интерактивная таблица):

| Дата | Оппонент | Счёт | Мой приём | Результат моего удара | Приём оппонента | Результат удара оппонента | → |
|---|---|---|---|---|---|---|---|

- Сортировка по каждому столбцу.
- Фильтрация по каждому столбцу (техника, результат удара, оппонент, дата и др.).
- Клик `→` → переход к видео.
- **Все блоки дашборда реагируют на активные фильтры таблицы.**

---

### 5.5. Экран просмотра видео

**Панель разметки (слева)**:
- Два дропдауна: Боец A / Боец B (предзаполнены из метаданных видео).
- Кнопка `START` (зелёная) — фиксирует начало схода в текущей позиции видео.
- Кнопка `FINISH` (красная) — фиксирует конец схода.
- Скроллируемый список карточек сходов в хронологическом порядке.

**Карточка схода (свёрнута)**:
`Сход N (HH:MM:SS — HH:MM:SS)`, счёт `A : B`

**Карточка схода (развёрнута)**:
- Для каждого бойца: очки (+/−), техника (выбор из списка), зона поражения (выбор), результат (попал / промахнулся / заблокировали).
- Отдельные кнопки `Сохранить` и `Свернуть`.
- При раскрытии другой карточки: текущая закрывается. Если есть несохранённые данные — диалог подтверждения.

**Футер панели**: `TOTAL SCORE A : B`.

**Видеоплеер (центр)**:
- Видео занимает максимально доступную площадь.
- Клик — play/pause. Колесо мыши — цифровой зум с индикатором в углу.

**Чат (справа)**:
- Сообщения: `Аватар | Имя | Тайм-код | Текст | [Ответить]`.
- Ответы смещены на 16px вправо.
- Клик на тайм-код → перемотка видео на этот момент и пауза.
- Поле ввода + Enter для отправки. Сообщение привязывается к текущей позиции видео.
- Все комментарии видят все пользователи.

**Таймлайн (снизу, на всю ширину экрана)**:

```
[ Метки комментариев (точки/иконки по хронологии)    ]
[ Основной прогресс-бар (клик / drag = перемотка)    ]
[ Трек сходов (цветные прямоугольники-сегменты)      ]
[ PLAY | ◀ | ▶ | LOOP | SPEED 0.1×–2.0× | VOL | TIME ]
```

- Клик на сегмент схода → устанавливает loop-диапазон + включает loop.
- Клик на метку комментария → перемотка видео.

**Горячие клавиши**:
- `Space` — пауза
- `X` — шаг вперёд на кадр
- `Z` — шаг назад на кадр

## 6. Нефункциональные требования

- Платформа: TrueNAS, одна установка.
- Несколько пользователей могут одновременно работать с одним видео (разметка + чат).
- Видео стримится из Seafile, не кешируется на сервере приложения.
