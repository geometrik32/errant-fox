# Техническое задание: Errant Fox

## 1. Обзор системы

Веб-приложение для видеоанализа тренировочных спаррингов по HEMA (длинный меч).
Одна установка на домашнем сервере TrueNAS. Все видео хранятся в Seafile.

---

## 2. Пользователи и права

| Возможность | Пользователь | Admin |
|---|---|---|
| Просмотр видео, статистики, чата | ✓ | ✓ |
| Разметка сходов | ✓ | ✓ |
| Комментарии в чате | ✓ | ✓ |
| Реакции (лайк/дизлайк) на комментарии | ✓ | ✓ |
| Поиск по комментариям | ✓ | ✓ |
| Редактирование своего профиля | ✓ | ✓ |
| Создание пользователей | — | ✓ |
| Редактирование пользователей | — | ✓ |
| Управление списком техник | — | ✓ |

**Профиль**: имя, аватар, пароль, цвет — пользователь меняет сам через ProfileModal. Admin может менять любые данные пользователей.
**Admin при создании**: логин, пароль, имя, аватар (опционально), флаг Admin, цвет (опционально).

---

## 3. Интеграция с Seafile

- Параметры подключения (`SEAFILE_URL`, `SEAFILE_TOKEN`) задаются один раз в `.env` при деплое.
- Приложение автоматически индексирует новые видео из Seafile каждые 60 секунд без участия пользователя.
- Структура: папки по датам тренировок → дата парсится из имени папки (regex `\d{4}-\d{2}-\d{2}`). Внутри — видеофайлы с произвольными именами (имена не используются нигде в UI).
- Видео стримится напрямую из Seafile, не кешируется на сервере.
- FPS видео извлекается из MP4 moov atom (первые 2 МБ) при синхронизации, без скачивания всего файла.
- Статичные превью-кадры (10 JPG) генерируются бэкендом через FFmpeg при первом запросе.

## 4. Модель данных

### Fighter (User)
`id`, `username`, `password_hash`, `display_name`, `avatar_path`, `color`, `is_admin`, `created_at`

### Technique
`id`, `name`, `description` — глобальный список, управляется Admin (добавить / изменить / удалить)

### HitZone (фиксированный enum — 16 зон)
Голова, Шея, Плечо пр., Предплечье пр., Кисть пр., Плечо лев., Предплечье лев., Кисть лев., Тело, Таз, Бедро пр., Голень пр., Стопа пр., Бедро лев., Голень лев., Стопа лев.

> Зоны записываются в формате `"Название:x:y"` где x,y — нормализованные координаты (0..1) точки удара на SVG-силуэте.

### Result (фиксированный enum — 7 значений)
`hit`, `miss`, `blocked`, `late`, `no_strike`, `disqualification`, `afterblow`

### Video
`id`, `seafile_path`, `fighter_a_id` (nullable), `fighter_b_id` (nullable), `date`, `duration_ms`, `preview_count`, `fps`, `created_at`

Итоговый счёт вычисляется динамически как сумма `score_a` и `score_b` всех сходов.

### Bout (Сход)
`id`, `video_id`, `time_start_ms`, `time_end_ms`, `order_index`

Боец A: `score_a` (int), `technique_a_id` (nullable), `hit_zone_a` (nullable), `result_a` (nullable)
Боец B: `score_b` (int), `technique_b_id` (nullable), `hit_zone_b` (nullable), `result_b` (nullable)

### Comment
`id`, `video_id`, `author_id`, `timestamp_ms`, `text`, `reply_to_id`, `bout_id`, `created_at`, `edited_at`

> `bout_id` определяется автоматически: если `timestamp_ms` попадает в диапазон схода — записывается его id.

### CommentReaction
`comment_id`, `user_id`, `kind` (`like` / `dislike`)

## 5. Экраны

### 5.1. Авторизация
Логотип, заголовок "Sign In", поля Username / Password, кнопка "Sign In".
Фон — тёмно-синий градиент. Регистрация отсутствует.

---

### 5.2. Глобальный Header
- **Лево**: логотип + "Errant Fox" → клик → галерея.
- **Центр**: переключатель `[Видео] [Бойцы]`.
- **Право**: иконка поиска (открывает SearchPanel), иконка темы (светлая/тёмная), иконка настроек + аватар → dropdown: профиль, выход; для Admin дополнительно: создать бойца, управление техниками.

---

### 5.3. Поиск по комментариям (SearchPanel)
- Поисковая строка: ввод текста → Enter → поиск через `GET /api/comments/search?q=...`
- Результаты: список с текстом комментария, автором, датой видео, именами бойцов, таймкодом
- Клик на результат → переход к видео на нужный момент

---

### 5.4. Экран «Видео» — Галерея

**Сайдбар (слева)**:
- Список чекбоксов бойцов с количеством видео. Мультивыбор.
- Виджет выбора диапазона дат.

**Сетка карточек (центр)**:
- Превью 16:9 (статичный кадр). При наведении — анимация scrub по превью-кадрам.
- **Размеченное видео**: `Боец A vs Боец B`, счёт `[X : Y]`, дата.
- **Неразмеченное видео**: на месте имён и счёта — визуальный маркер (иконка + "Заполните данные").
- Клик на неразмеченное → модальная форма: выбор Бойца A и Бойца B (дата уже проставлена из имени папки Seafile).
- Клик на размеченное → экран просмотра видео.

---

### 5.5. Экран «Бойцы» — Статистика

**Сайдбар (слева)**:
- Список имён бойцов для переключения дашбордов.

> Единственный фильтр данных — через таблицу "История боёв" (фильтры по столбцам). Все графики и силуэты реагируют на него.

**Дашборд бойца**:

**Header**: аватар + имя + дата первого зафиксированного боя.

**Блоки быстрой статистики**:
- Использую приём чаще всего: `[название техники]`
- Промахиваюсь чаще всего с приёмом: `[название техники]`
- Получаю урон чаще всего от приёма: `[название техники]`

**Радарная диаграмма результатов**: 7 осей — hit, afterblow, late, disqualification, no_strike, miss, blocked. Значения — процент от общего числа боёв.

**Блок «Топ техник»**: таблица техник с количеством использований, попаданий и процентом успешности.

**График — Частота поединков**: ось X — недели, ось Y — количество боёв.

**График — Динамика результатов**:
- Ось X — бои по хронологии, ось Y — победа вверх / поражение вниз (бинарно).
- Фильтр по конкретному противнику (дропдаун над графиком).
- В заголовке: общий процент побед и поражений.
- Победитель — у кого сумма очков за все сходы видео больше.

**График — Прогресс по баллам**: линейный график суммарных очков за тренировку по хронологии.

**Последние оппоненты**: список противников с количеством побед/поражений/всего боёв.

**Зона поражения — нанесённый урон**: силуэт человека в T-позе, интенсивность закраски 6 зон по частоте попаданий бойца.

**Зона поражения — полученный урон**: то же, но по ударам, пришедшим по бойцу.

> Силуэты динамически обновляются при фильтрации таблицы истории боёв.

**История боёв** (интерактивная таблица):

| Дата | Оппонент | Счёт | Мой приём | Результат моего удара | Приём оппонента | Результат удара оппонента | → |
|---|---|---|---|---|---|---|---|

- Сортировка по каждому столбцу.
- Фильтрация по каждому столбцу (техника, результат удара, оппонент, дата и др.).
- Клик `→` → переход к видео с авто-перемоткой на нужный таймкод.
- **Все блоки дашборда реагируют на активные фильтры таблицы.**

---

### 5.6. Экран просмотра видео

**Панель разметки (слева)**:
- Два дропдауна: Боец A / Боец B (предзаполнены из метаданных видео).
- Кнопка `START` (зелёная) — фиксирует начало схода в текущей позиции видео.
- Кнопка `FINISH` (красная) — фиксирует конец схода.
- Скроллируемый список карточек сходов в хронологическом порядке.

**Карточка схода (свёрнута)**:
`Сход N (HH:MM:SS — HH:MM:SS)`, счёт `A : B`

**Карточка схода (развёрнута)**:
- Для каждого бойца: очки (+/−), техника (выбор из списка), зона поражения (HitZonePicker — SVG-силуэт с 16 кликабельными зонами), результат (выбор из 7 значений).
- Отдельные кнопки `Сохранить` и `Свернуть`.
- При раскрытии другой карточки: текущая закрывается.

**Футер панели**: `TOTAL SCORE A : B`.

**Видеоплеер (центр)**:
- Видео занимает максимально доступную площадь.
- Клик — play/pause. Колесо мыши — цифровой зум с индикатором в углу.

**Чат (справа)**:
- Сообщения: `Аватар | Имя | Тайм-код | Текст | [Ответить]`.
- Кнопки лайк/дизлайк с количеством.
- Ответы смещены на 16px вправо.
- Клик на тайм-код → перемотка видео на этот момент и пауза.
- Поле ввода + Enter для отправки. Сообщение привязывается к текущей позиции видео и автоматически к сходу.
- Все комментарии видят все пользователи.

**Таймлайн (снизу, на всю ширину экрана)**:

```
[ Метки комментариев (точки/иконки по хронологии)    ]
[ Основной прогресс-бар (клик / drag = перемотка)    ]
[ Трек сходов (цветные прямоугольники-сегменты)      ]
[ PLAY | ◀ | ▶ | LOOP | SPEED 0.1×–2.0× | VOL | TIME ]
```

- Клик на сегмент схода → устанавливает loop-диапазон + включает loop.
- Клик на метку комментария → перемотка видео.

**Горячие клавиши**:
- `Space` — пауза
- `X` — шаг вперёд на кадр
- `Z` — шаг назад на кадр

---

## 6. Нефункциональные требования

- Платформа: TrueNAS SCALE, одна установка.
- Несколько пользователей могут одновременно работать с одним видео (разметка + чат).
- Видео стримится из Seafile, не кешируется на сервере приложения.
- Тёмная и светлая темы (переключаются в Header).
- Адаптивная вёрстка: при `max-width: 768px` — колоночная раскладка, скрытие боковых панелей.
