# Техническое задание: Errant Fox

## 1. Обзор системы

Веб-приложение для видеоанализа тренировочных спаррингов по HEMA (длинный меч).
Развертывание на домашнем сервере TrueNAS в Docker. Все видео хранятся в Seafile.
Платформа включает модуль авторазметки сходов на базе искусственного интеллекта (Faster-Whisper + акустическая коррекция пиков выкриков) и систему публичного шаринга видео.

---

## 2. Пользователи и права

В системе предусмотрено 3 роли: **Admin** (тренер / администратор), **Fighter** (фехтовальщик) и **Guest** (гость по внешней ссылке).

| Возможность | Guest | Fighter | Admin |
|---|---|---|---|
| Просмотр расшаренного видео и сходов | ✓ | ✓ | ✓ |
| Комментирование (с вводом никнейма) | ✓ | ✓ | ✓ |
| Просмотр всей галереи и статистики | — | ✓ | ✓ |
| Ручная разметка сходов | — | ✓ | ✓ |
| Полнотекстовый поиск по комментариям | — | ✓ | ✓ |
| Запуск ИИ-разметки сходов (одиночный и пакетный) | — | — | ✓ |
| Отмена ИИ-разметки | — | — | ✓ |
| Управление пользователями и техниками | — | — | ✓ |

---

## 3. Интеграция с Seafile и ИИ-модулем

- **Seafile**: Автоматический фоновый опрос папок тренировок каждые 60 секунд. Извлечение FPS из moov atom и прямая передача ссылок воспроизведения клиенту и ИИ-микросервису.
- **Whisper Service (ИИ-авторазметка)**:
  - Автоматическое выявление судейских команд и стоп-слов («Стоп», «Бой», «Время», «Хоп» и др.).
  - **Acoustic Peak Refinement**: Алгоритм уточнения границ схода по физическому пику громкости акустической волны в диапазоне выкрика.
  - Пакетная разметка (`Batch AI Modal`): Администратор может запустить ИИ-анализ всех выделенных или всех неразмеченных видео в один клик.
  - Мгновенная отмена: При отмене ИИ-анализа микросервис безотлагательно вычеркивает задачу из очереди и прекращает транскрипцию.

## 4. Модель данных

### User (Пользователь)
`id`, `username`, `password_hash`, `display_name`, `avatar_path`, `color`, `is_admin`, `created_at`, `vk_id` (nullable), `role` (`admin` / `fighter` / `guest`)

### Video (Видеозапись)
`id`, `seafile_path`, `fighter_a_id` (nullable), `fighter_b_id` (nullable), `date`, `duration_ms`, `preview_count`, `fps`, `created_at`, `is_ai_labeled` (bool), `is_analyzing` (bool)

### Bout (Сход)
`id`, `video_id`, `time_start_ms`, `time_end_ms`, `order_index`, `score_a`, `score_b`, `technique_a_id`, `technique_b_id`, `hit_zone_a`, `hit_zone_b`, `result_a`, `result_b`, `is_ai` (bool)

### BoutHistory (История изменений схода)
`id`, `bout_id`, `user_id`, `action` (`create`, `update`, `delete`, `ai_generate`), `details`, `created_at`

### Comment (Комментарий)
`id`, `video_id`, `author_id`, `timestamp_ms`, `text`, `reply_to_id`, `bout_id`, `guest_nickname` (nullable), `created_at`, `edited_at`

### CommentReaction (Реакция)
`comment_id`, `user_id`, `kind` (`like` / `dislike`)

## 5. Экраны и Интерактивные компоненты

### 5.1. Галерея видео
- Карточки с покадровой анимацией превью при наведении (scrubbing).
- Маркеры статуса разметки: "Размечено человеком", "Размечено ИИ", "Выполняется анализ ИИ..." (с пульсирующим лоадером).
- Кнопка вызова пакетной ИИ-разметки для неразмеченных видео.

### 5.2. Пакетная ИИ-разметка (Batch AI Modal)
- Отображает список выбранных видео для обработки.
- Показывает прогресс выполнения в реальном времени через WebSocket.
- Кнопка отмены для остановки очереди или текущей видеозаписи.

### 5.3. Видеоплеер и Судейская панель
- Покадровый просмотр (клавиши Z/X), цифровой зум (колесо мыши), зацикливание сходов (Loop).
- Судейская панель: быстрый ввод баллов, 16 зон попадания (SVG HitZonePicker), результаты удара.
- Метка у схода, если он был автоматически сгенерирован ИИ.

### 5.4. Встроенный чат и Публичный Шаринг
- Отправка комментариев к конкретным таймкодам.
- Кнопка "Поделиться" в шапке плеера: генерация публичной ссылки с опциональной привязкой к конкретному сходу.
- Страница публичного просмотра для гостей без аккаунта.

---

## 6. Нефункциональные требования

- Развертывание в Docker Compose (Rust бэкенд, Python ИИ-сервис, Nginx фронтенд).
- Микросервис Whisper AI оптимизирован для выполнения на обычных процессорных инструкциях CPU (int8 CTranslate2) без строгой необходимости в GPU CUDA.
- Ограничение нагрузки ИИ: последовательная обработка в очереди с контролем использования ОЗУ и дискового пространства.
