# Дизайн-код платформы Errant Fox

Данный документ описывает единые правила визуального оформления, иерархию элементов, дизайн-токены и принципы компоновки интерфейса для системы видеоанализа спаррингов HEMA — **Errant Fox**.

---

## 1. Концепция интерфейса: «Островной» стиль (Island Layout)

Интерфейс Errant Fox строится на концепции **островных полупрозрачных блоков** (панелей), парящих над глубоким базовым фоном страницы. Каждый функциональный блок является изолированным визуальным контейнером.

### Основные принципы компоновки:
1. **Базовый слой:** Единый глубокий тёмный фон страницы (`#0F172A`).
2. **Панели-острова:** Вся функциональная разметка (шапка, боковые панели, сетки контента, модули плеера) оформляется в виде карточек с полупрозрачным фоном и тонкими рамками.
3. **Фиксированные отступы (Gaps):** Расстояние между соседними островами строго зафиксировано и составляет **16px** (или **24px** на разреженных страницах вроде галереи).
4. **Внутренние отступы (Padding):** Внутри панелей отступы составляют **24px** для крупных блоков и **16px** для компактных панелей управления.

---

## 2. Цветовая палитра (Color Tokens)

Платформа работает исключительно в тёмной теме. Оттенки серого и синего минимизируют утомляемость глаз при многократном пересмотре видеоматериалов.

### Базовые фоны
* **Базовый фон страницы (`--bg-base`):** `#0F172A` — глубокий сине-черный цвет.
* **Поверхность панели (`--surface`):** `rgba(31, 41, 55, 0.65)` — полупрозрачный тёмный графит.
* **Сплошная панель (`--surface-solid`):** `#1F2937` — для выпадающих меню и всплывающих модальных окон.
* **Элемент при наведении (`--surface-hover`):** `#374151` — выделение кликабельных интерактивных областей.
* **Глубокая подложка (`--surface-deep`):** `rgba(55, 65, 81, 0.85)` — для выделенных или заблокированных зон.

### Акцентные и семантические цвета
* **Основной акцент (`--accent-yellow`):** `#F59E0B` (активный ховер: `#D97706`) — используется для ключевого фокуса (таймлайн, активные табы, счет боя, индикация зума).
* **Успех / Захват (`--accent-green`):** `#10B981` — индикация начала разметки схода (START), статус бойца «в сети».
* **Ошибка / Удаление (`--accent-red`):** `#E05252` — границы окончания схода (FINISH), сообщения об ошибках, кнопки удаления, выход из аккаунта.
* **Интеллектуальный помощник (`--accent-purple`):** `#8E44AD` — операции искусственного интеллекта (разметка сходов ИИ, индикатор анализа).
* **Вспомогательный акцент (`--accent-blue`):** `#1FA5D8` — ссылки общего доступа, системные информационные баннеры.

### Границы и разделители
* **Тонкая граница (`--border-color`):** `rgba(255, 255, 255, 0.05)` — стандартное очерчивание островов и разделителей.
* **Контрастная граница (`--border-strong`):** `rgba(255, 255, 255, 0.1)` — для полей ввода в фокусе или активных кнопок.

## 3. Типографика и иерархия шрифтов

Шрифт по умолчанию: **'Inter', sans-serif** (подключается через Google Fonts с начертаниями 300, 400, 500, 600, 700).

| Уровень и роль | Размер шрифта (CSS) | Вес (Weight) | Цвет текста | Применение |
| :--- | :--- | :--- | :--- | :--- |
| **H1 (Page Header)** | `32px` (`2rem`) | `700` (Bold) | `--text-primary` | Заголовки страниц авторизации, логотип |
| **H2 (Section Header)** | `22px` (`1.4rem`) | `700` (Bold) | `--text-primary` | Заголовки модальных окон, крупные разделы |
| **H3 (Block Header)** | `13px` (`0.8rem`) | `600` (SemiBold) | `--text-secondary` | Заголовки списков, категорий (всегда `uppercase`, `letter-spacing: 0.09em`) |
| **Fighter Name** | `15px` (`0.95rem`) | `600` (SemiBold) | `--text-primary` | Имена бойцов в списках, карточках, чате |
| **Body Text** | `14px` (`0.85rem`) | `400` / `500` | `--text-secondary` | Описания, комментарии, тексты настроек |
| **Meta / Small** | `12px` (`0.75rem`) | `400` (Regular) | `--text-muted` | Юзернеймы (`@login`), счетчики видео, таймкоды |
| **Control Labels** | `13px` (`0.8rem`) | `700` (Bold) | `--text-primary` | Текст на кнопках управления (START, LOOP) |

## 4. Скругления углов (Border Radius)

Геометрия платформы сбалансированная, умеренно скругленная. Экстремально круглые формы (pill) используются только для индикаторов статуса и переключателей навигации.

* **Панели и Модальные окна (`--radius-lg`):** `16px` — внешние контуры островов (плеер, сайдбары, модалки).
* **Внутренние карточки (`--radius-md`):** `12px` — карточки видео, карточка профиля бойца, выпадающие списки.
* **Кнопки и Поля ввода (`--radius-sm`):** `8px` — инпуты, кнопки управления плеером, кнопки фильтров.
* **Круглые элементы (`--radius-pill`):** `9999px` (или `50%`) — аватары, индикаторы присутствия, ползунки, переключатели вкладок.

## 5. Иерархия аватаров и иконок бойцов

Иконки бойцов всегда снабжаются цветной рамкой-обводкой, цвет которой соответствует персональному цвету бойца.

1. **Large Avatar (Профиль / Герой статистики):**
   * **Размер:** `90px` на `90px`.
   * **Оформление:** Скругление `50%`, толщина рамки `3px`, цвет рамки соответствует цвету бойца.
2. **Medium Watcher (Аватары зрителей в плеере):**
   * **Размер:** `36px` на `36px`.
   * **Оформление:** Скругление `50%`, толщина рамки `1.5px` (белая/акцентная).
3. **Standard Avatar (Сайдбар бойцов, шапка профиля, чат):**
   * **Размер:** `32px` на `32px`.
   * **Оформление:** Скругление `50%`, толщина рамки `2px`, цвет рамки соответствует цвету бойца.
4. **Small Avatar (Присутствие на таймлайне):**
   * **Размер:** `24px` на `24px`.
   * **Оформление:** Скругление `50%`, толщина рамки `1px`.

## 6. Компоненты управления датами: Единый Календарь

Календарь (компонент `DateRangePicker`) поддерживает два режима отображения, переключаемых в шапке выпадающего окна:

1. **Понедельный (Weekly):**
   * Вместо сетки из 31 дня отображается список недель текущего месяца.
   * Описание строки: `[Дата начала недели] — [Дата окончания недели]` (например, `12 Янв — 18 Янв`).
   * Рядом с неделей выводится точка-маркер присутствия видео спаррингов.
   * Выбор недели кликом устанавливает диапазон дат с понедельника по воскресенье выбранной недели.
2. **Классический (Classic):**
   * Стандартная сетка дней месяца.
   * Выбор произвольного диапазона в два клика (выбор даты «С» и даты «По» с закрашиванием промежуточного интервала).
