Web application for HEMA (Historical European Martial Arts, longsword only) sparring video analysis. Single installation on a home TrueNAS server.

**Why:** Personal club tool for fighters to review sparring footage, tag exchanges, and track performance statistics.

**How to apply:** All decisions should assume single-tenant, single-club deployment. No multi-tenancy, no SaaS complexity.

## Platform & Storage
- Web app, deployed on TrueNAS home server
- Videos stored in Seafile cloud (on same server); credentials set once in deploy config (URL + API token)
- Video streams directly from Seafile — backend only issues a temporary URL, video bytes never pass through our backend
- Seafile folder structure: folders named by training date → video files inside (arbitrary names, never shown in UI)
- Preview frames generated by backend via FFmpeg (not Seafile API), cached on disk

## Users & Auth
- No self-registration. Admin creates all accounts.
- Two roles: regular user and Admin. All users equal except Admin can: create users, manage technique list.
- Users can change: own name, avatar, password. Cannot change admin status.

## Tech Stack
- **Backend:** Rust + Axum + SQLite (Diesel ORM)
- **Frontend:** Svelte 5 + TypeScript + Vite (taken from TrueNAS prototype)
- **Video player:** HybridVideoDecoder (HTML5 + WebCodecs via Mediabunny) — already built in TrueNAS prototype
- **API:** REST (JSON) for data + WebSocket (JSON) for 3 live events only
- **Auth:** JWT (7-day) + bcrypt
