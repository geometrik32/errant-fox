# Директива: Дизайн Frontend (Errant Fox)

## Цель
Полный редизайн интерфейса приложения Errant Fox с использованием стиля, предоставленного в референсе (мягкие тени, скругления, glassmorphism, желтые акценты). Реализация переключения между светлой и темной темой, а также полная адаптация под мобильные устройства.

## Входные данные
- Существующие компоненты Svelte в `frontend/src/` (функционал компонентов не изменяется).
- Референсное изображение (предоставлено пользователем).
- Стандартный стек: HTML, Vanilla CSS, Svelte 5.

## Требования к стилю (Design Tokens)
1. **Шрифты**: Использовать `Inter` (Google Fonts).
2. **Формы**: Закругленные углы для карточек (`border-radius: 20px` или `24px`) и кнопок (`border-radius: 9999px`).
3. **Эффекты**: Использование `backdrop-filter: blur(10px)` для эффекта стекла, мягкие тени `box-shadow: 0 8px 32px rgba(0,0,0,0.05)`.
4. **Светлая тема (default)**:
   - Фон: градиент от белого/кремового к светло-желтому.
   - Текст: темно-серый/черный.
   - Карточки: белые с прозрачностью.
5. **Темная тема**:
   - Фон: градиент темно-серый/черный.
   - Текст: светло-серый/белый.
   - Карточки: темно-серые с прозрачностью.
6. **Адаптивность**: При `max-width: 768px` элементы выстраиваются в одну колонку (flex-direction: column), скрываются боковые панели в гамбургер-меню (если применимо), уменьшаются отступы.

## Инструменты
- Использование CSS переменных (`var(--bg-gradient)`, `var(--card-bg)`, и т.д.) в `app.css`.
- Обертывание логики переключения в `data-theme="dark"` атрибут на теге `<html>` или `<body>`.

## Выходные данные
- Обновленные файлы `*.svelte` с новыми CSS-стилями.
- Глобальный файл `app.css` с дизайн-токенами.
- Обновленный `index.html`.

## Edge Cases (Особые случаи)
- Компоненты, которые генерируются динамически (графики Chart.js), должны наследовать цвета темы.
- При переключении темы, графики могут потребовать перерисовки (вызов `.update()`), либо использования универсальных цветов, которые хорошо выглядят на обоих фонах.
